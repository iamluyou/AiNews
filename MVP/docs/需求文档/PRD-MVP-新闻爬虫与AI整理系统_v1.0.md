# MVP 新闻爬虫与 AI 整理系统 PRD

## 1. 项目概述

### 1.1 项目背景
构建一个自动化的新闻采集、AI 整理与分发系统，定时从多个 AI 相关网站爬取新闻，通过大模型进行智能筛选和排序，并通过多种渠道（飞书、邮件）分发。

### 1.2 项目目标
- 自动化采集指定网站的 AI 相关新闻
- 通过大模型进行智能筛选和排序
- 多渠道定时推送整理后的新闻

### 1.3 目标用户
- AI 行业从业者
- 关注 AI 动态的用户群体

---

## 2. 功能需求

### 2.1 新闻爬虫模块

| ID | 功能点 | 详细说明 |
|----|--------|---------|
| F1.1 | 多网站支持 | 支持从以下网站爬取新闻：<br>- https://www.jiqizhixin.com/<br>- https://www.36kr.com/information/AI/<br>- https://aiera.com.cn/<br>- https://radarai.top/<br>- https://www.qbitai.com/ |
| F1.2 | 信息提取 | 提取新闻以下字段：<br>- 标题<br>- 链接<br>- 新闻封面图<br>- 发布时间<br>- 来源<br>- 内容 |
| F1.3 | 定时执行 | 每天定时执行三次：08:30、11:30、17:30 |

### 2.2 数据存储模块

| ID | 功能点 | 详细说明 |
|----|--------|---------|
| F2.1 | 数据库存储 | 将爬取的新闻存储到数据库 |
| F2.2 | 字段定义 | 数据库包含字段：<br>- 标题<br>- 链接<br>- 新闻封面图<br>- 发布时间<br>- 来源<br>- 内容 |
| F2.3 | 排序功能 | 按发布时间对新闻列表进行排序 |

### 2.3 飞书通知模块

| ID | 功能点 | 详细说明 |
|----|--------|---------|
| F3.1 | Webhook 推送 | 每次爬取后通过飞书 Webhook 发送到飞书群 |
| F3.2 | Webhook 配置 | Webhook 地址在配置文件中管理 |

### 2.4 大模型整理模块

| ID | 功能点 | 详细说明 |
|----|--------|---------|
| F4.1 | AI 整理 | 调用大模型接口整理新闻内容 |
| F4.2 | 智能排序 | 按 AI 相关度排序，相关度越高越靠前 |
| F4.3 | 去重/去无效 | 去除无效信息和重复内容 |
| F4.4 | 批量调用 | 支持批量处理多个文本 |
| F4.5 | 模型选择 | 支持自定义选择不同的大模型 |

### 2.5 邮件通知模块

| ID | 功能点 | 详细说明 |
|----|--------|---------|
| F5.1 | 163 邮件发送 | 通过 163 邮箱发送整理后的新闻 |
| F5.2 | 指定收件人 | 支持配置指定的收件邮箱 |

### 2.6 配置管理模块

| ID | 功能点 | 详细说明 |
|----|--------|---------|
| F6.1 | 统一配置 | Key、URL 等敏感信息统一在配置文件管理 |
| F6.2 | 快速替换 | 支持部署时快速替换配置 |

## 3. 非功能需求

| 类别 | 需求描述 |
|------|---------|
| 性能 | 单次爬取任务完成时间 < 5 分钟 |
| 可靠性 | 爬虫失败后应有重试机制 |
| 可扩展性 | 支持方便地添加新的爬取网站 |
| 可维护性 | 代码结构清晰，有完善的日志记录 |
| 安全性 | 敏感信息（Key、Webhook 等）不硬编码 |

---

## 4. 数据流程

```
定时触发 → 多网站爬虫 → 数据提取 → 数据库存储
                                              ↓
飞书群通知 ← 发送 Webhook ← 新闻列表 ← 按时间排序
                                              ↓
邮件发送 ← 163 邮件 ← AI 排序整理 ← 大模型接口
```

## 5. 技术选型建议

| 模块 | 技术选型 |
|------|---------|
| 开发语言 | Python 3.10+ |
| 定时任务 | APScheduler / Celery Beat |
| 爬虫框架 | requests + BeautifulSoup / Playwright |
| 数据库 | SQLite（MVP）/ PostgreSQL（后续） |
| ORM | SQLAlchemy / Peewee |
| 大模型 SDK | OpenAI SDK（兼容通用协议） |
| 邮件发送 | smtplib / yagmail |
| 配置管理 | Pydantic Settings / python-dotenv |
| 日志 | loguru / logging |

---

## 6. 验收标准

1. 能够成功从 5 个指定网站爬取新闻列表
2. 每天 08:30、11:30、17:30 准时执行任务
3. 飞书群能收到每次爬取的新闻通知
4. AI 整理后的新闻按相关度排序，去除无效信息
5. 指定邮箱能收到整理后的新闻邮件
6. 所有配置项可通过配置文件修改

---

## 7. 风险与注意事项

| 风险 | 应对措施 |
|------|---------|
| 网站反爬虫 | 合理设置请求间隔，使用 User-Agent 轮换 |
| 网站结构变更 | 各网站爬虫独立封装，便于单独维护 |
| 大模型 API 限流 | 实现批量调用和重试机制 |
| 邮件/飞书发送失败 | 记录失败日志，支持手动重发 |
