# MVP 新闻爬虫与 AI 整理系统 - 技术设计文档

## 1. 架构概述

### 1.1 系统架构

```
┌─────────────────────────────────────────────────────────────┐
│                        Scheduler (APScheduler)              │
│                    (定时任务调度，可配置时间点)             │
└────────────────────────────┬────────────────────────────────┘
                             │
        ┌────────────────────┼────────────────────┐
        │                    │                    │
        ▼                    ▼                    ▼
┌───────────────┐   ┌───────────────┐   ┌───────────────┐
│   Crawlers    │   │   Storage     │   │   Notifiers   │
│  (多网站并发)  │   │  (SQLite)    │   │ (飞书/邮件)   │
└───────┬───────┘   └───────┬───────┘   └───────────────┘
        │                    │
        ▼                    ▼
┌───────────────┐   ┌───────────────┐
│     LLM       │   │   Repository  │
│  (OpenAI API) │   │  (批量操作)   │
│  (带重试)     │   └───────────────┘
└───────────────┘
        │
        ▼
┌───────────────┐
│   Web UI      │
│   (Flask)     │
└───────────────┘
```

### 1.2 核心设计原则

1. **模块化设计**：各功能模块独立，通过接口交互
2. **可扩展性**：基于基类抽象，易于扩展新的爬虫、通知渠道、LLM 提供商
3. **配置驱动**：所有关键配置通过配置文件管理
4. **可测试性**：核心逻辑独立，便于单元测试
5. **高性能**：并发爬虫 + 批量数据库操作
6. **容错性**：完善的降级机制和错误处理

## 2. 模块详细设计

### 2.1 爬虫模块 (Crawlers)

**基类设计**：
```python
class BaseCrawler(ABC):
    name: str
    base_url: str
    source_name: str

    @abstractmethod
    def fetch() -> List[NewsItem]:
        pass
```

**并发执行设计**：
- 使用 `ThreadPoolExecutor` 实现并发
- 配置项 `crawlers.max_concurrent` 控制并发数（默认 3）
- 单个爬虫失败不影响其他爬虫
- 支持回退到串行执行

**扩展方式**：
- 继承 `BaseCrawler`
- 实现 `fetch()` 方法
- 在 `CRAWLER_REGISTRY` 中注册

**当前支持爬虫**：
- kr36: 36氪
- aiera: Aiera
- radar: RadarAI
- qbit: QbitAI

### 2.2 通知模块 (Notifiers)

**基类设计**：
```python
class BaseNotifier(ABC):
    name: str

    @abstractmethod
    def send(news_list: List[NewsItem], title: str, **kwargs) -> bool:
        pass
```

**扩展方式**：
- 继承 `BaseNotifier`
- 实现 `send()` 方法

### 2.3 大模型模块 (LLM)

**基类设计**：
```python
class BaseLLM(ABC):
    name: str

    @abstractmethod
    def chat(messages: List[dict]) -> Optional[str]:
        pass

    @abstractmethod
    def batch_chat(prompts: List[str]) -> List[Optional[str]]:
        pass

    def process_news(news_list: List[NewsItem]) -> List[NewsItem]:
        # 去重、评分、排序
        pass
```

**重试机制设计**：
- 指数退避重试：`2 ** attempt` 秒
- 可配置最大重试次数
- 详细的重试日志

**降级方案**：
- LLM 失败时使用关键词评分
- 关键词：AI、人工智能、大模型、GPT、LLM、机器学习、深度学习、Agent、Claude、Gemini、Sora、文心、通义、智谱

**预留 RAG 接口**：
- 未来可在 `process_news()` 中接入 RAG 检索
- 支持多 RAG 源配置

### 2.4 存储模块 (Storage)

**数据模型**：
- 使用 SQLAlchemy ORM
- SQLite（MVP）→ PostgreSQL（后续）
- Repository 模式封装数据访问

**批量操作优化**：
- `add_batch()`: 单 session 批量 upsert
- `mark_as_sent()`: 批量更新
- 失败时自动降级到单条操作

**Repository 模式**：
```python
class NewsRepository:
    @staticmethod
    def add(news: NewsItem) -> bool
    @staticmethod
    def add_batch(news_list: List[NewsItem]) -> int
    @staticmethod
    def mark_as_sent(urls: List[str]) -> int
    @staticmethod
    def is_sent(url: str) -> bool
    # ... 其他查询方法
```

### 2.5 工具模块 (Utils)

**去重工具**：
```python
def deduplicate_news_by_url(news_list: List[NewsItem]) -> List[NewsItem]
```
- 按 URL 去重，保持顺序
- 统一 scheduler 和 llm 模块的去重逻辑

### 2.6 配置模块 (Config)

**配置验证设计**：
- 使用 Pydantic Validators
- 验证项：
  - cron 时间格式 (HH:MM)
  - URL 格式 (http/https)
  - 邮箱格式
  - 日志级别
  - 数值范围（timeout、retries、并发数等）

**配置项结构**：

## 3. 数据流程

```
1. 定时触发
   ↓
2. 并发执行所有爬虫 → 收集 NewsItem 列表
   ↓
3. 按 URL 内存去重
   ↓
4. 按发布时间排序
   ↓
5. 批量存储到数据库
   ↓
6. 过滤已发送新闻
   ↓
7. 飞书通知（整理后的新闻）
   ↓
8. LLM 处理（去重、AI 相关度评分、排序）
   ↓
9. 邮件通知（整理后的新闻）
   ↓
10. 批量标记已发送
```

## 4. 性能优化设计

### 4.1 并发爬虫
- ThreadPoolExecutor 实现并发
- 可配置最大并发数
- 失败隔离，单个爬虫失败不影响其他

### 4.2 批量数据库操作
- 单 session 内完成批量操作
- 先查询已存在记录，再分别处理更新和插入
- 使用 bulk_save_objects 批量插入
- 失败时自动降级到单条操作

### 4.3 去重优化
- 内存去重（URL）+ 数据库去重（URL 主键）
- 统一去重工具函数

## 5. 扩展性设计

### 5.1 新增爬虫网站

1. 在 `crawlers/` 下创建新文件
2. 继承 `BaseCrawler`
3. 实现 `fetch()` 方法
4. 在 `crawlers/__init__.py` 的 `CRAWLER_REGISTRY` 中注册
5. 在配置文件 `crawlers.enabled` 中启用

### 5.2 新增通知渠道

1. 在 `notifiers/` 下创建新文件
2. 继承 `BaseNotifier`
3. 实现 `send()` 方法
4. 在 `scheduler.py` 中初始化并添加到 `notifiers` 列表

### 5.3 新增 LLM 提供商

1. 在 `llm/` 下创建新文件
2. 继承 `BaseLLM`
3. 实现 `chat()` 和 `batch_chat()` 方法
4. 在 `scheduler.py` 中根据配置选择使用

### 5.4 接入 RAG

预留扩展点：
- 在 `BaseLLM.process_news()` 中增加 RAG 检索步骤
- 配置文件增加 `rag` 配置段
- 支持多 RAG 源配置

## 6. 技术选型说明

| 组件 | 选型 | 说明 |
|-----|------|------|
| 定时任务 | APScheduler | 功能强大，支持动态修改 |
| 爬虫 | requests + BeautifulSoup | 简单易用，适合静态页面 |
| 并发 | concurrent.futures | Python 标准库，稳定可靠 |
| 数据库 | SQLite | MVP 阶段轻量化选择 |
| ORM | SQLAlchemy | Python 主流 ORM |
| LLM | OpenAI SDK | 兼容通用协议 |
| 邮件 | yagmail | 简单易用的邮件库 |
| 配置 | Pydantic Settings + YAML | 类型安全，易于管理 |
| 配置验证 | Pydantic Validators | 声明式验证，清晰易用 |
| 日志 | loguru | 简单强大的日志库 |
| Web UI | Flask | 轻量级 Web 框架 |
| 测试 | pytest | Python 主流测试框架 |
